Chicken Scratch — How It Works

A field sketch becomes clean text or a dimensioned blueprint, end to end.

1

Upload

UploadPanel.tsx
  • Pick a model — currently just gemini-flash-latest (the only entry in MODEL_OPTIONS)
  • Upload an image via camera or file input
  • Client-enforced 10 MB size limit
↓
2

Client orchestration

ChickenScratchApp.tsx
  • Base64-encode the image
  • State transitions idle → processing
  • Calls supabase.functions.invoke("process-scratch", { image, mimeType, model })
↓
3

Edge function

supabase/functions/process-scratch/index.ts
  • Validate image MIME type + model against ALLOWED_MODELS
  • Hash image + "::v3" (current PROMPT_VERSION) → look up scratch_cache by (image_hash, model)

Cache hit

Return the cached ProcessResult directly — no model call.

Cache miss

  1. Call Gemini (GoogleGenerativeAI, gemini-flash-latest, GEMINI_API_KEY) with PROMPT
  2. If result type === "blueprint", run a second-pass verification via multi-turn chat with VERIFY_PROMPT
  3. Write the result to scratch_cache
  • Either way, returns a ProcessResult
↓
branch on ProcessResult.type
type: "handwriting"

4a. Handwriting result

  • { type: "handwriting", markdown: string }
↓

5a. Render markdown

ResultPanel.tsx
  • Renders the markdown via marked.js
  • State processing → done (or error on failure)
type: "blueprint"

4b. Blueprint result

  • { type: "blueprint", elements: Shape[], labels: DimensionLabel[] }
↓

5b. Render to canvas

BlueprintRenderer.tsx / blueprintDraw.ts
  • Canvas2D raster — MODEL_SIZE = 1000, RENDER_SCALE = 2
  • centeringOffset() centers the shapes' bounding box + dimension margins in the grid
  • Fixed technical palette (white bg, charcoal ink, gray dimension lines), independent of app theme
↓
6

Output actions

ResultPanel.tsx
  • Copy / Download — handwriting as Markdown text / .md; blueprint as PNG via canvasToBlob / canvasToDataUrl (clipboard falls back to download)
  • Send to DohDocs — createDoc() → saveDoc(), then navigate to /dashboard/app/tasks. Blueprint is embedded as a base64 PNG (![Blueprint](data:image/png;...)) with dimension labels appended as a Markdown list

State machine

idle → processing → done
idle → processing ↘ error
"Try Again" / "New Image" resets done or error back to idle. There's no edge-function retry — the user just re-uploads.

Gotchas & ops notes

  • FunctionsHttpError wraps the real error in .context (a Response) — call .context.json() for { error }; logging the raw error won't show it.
  • Bump PROMPT_VERSION whenever PROMPT / VERIFY_PROMPT change — it salts the scratch_cache hash, so stale extractions stop being served.
  • Edits to index.ts require manual supabase functions deploy process-scratch — not part of the normal Vercel deploy.
